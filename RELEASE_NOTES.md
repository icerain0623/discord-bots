# Release Notes

## v0.15.0 — タスク権限をロールベースに変更

> 2026-03-27

### 変更

- `/task allow-user` → `/task allow-role` — ロール単位でタスク追加を許可
- `/task remove-user` → `/task remove-role` — ロールの許可を取り消し
- `/task allowed-users` → `/task allowed-roles` — 許可ロール一覧を表示
- 権限チェックをユーザーID判定からロール判定に変更（`interaction.member.roles` を使用）
- KVデータ構造を `allowedUsers[]` → `allowedRoles[]` に変更（既存データは自動マイグレーション）

## v0.14.0 — タスク管理機能

> 2026-03-27

### 新機能

- `/task add` — タスクを追加（名前・期限・優先度）
- `/task list` — タスク一覧を表示
- `/task complete` — タスクを完了にする

- 権限モデル: add はモデレーター（MANAGE_MESSAGES）または許可ロール、complete/delete/config はサーバー管理者（MANAGE_GUILD）、list は制限なし

## v0.12.0 — 1文リレーイベント機能

> 2026-03-23

### 新機能

- `/relay` コマンド — サーバーイベント用の1文リレー機能
  - `/relay start <topic> <first_sentence>` — お題と最初の一文を設定してリレー開始
  - `/relay status` — 全文を番号付きで表示（管理者のみ、エフェメラル）
  - `/relay delete <number>` — 指定番号の文を削除（不適切な内容の管理用）
  - `/relay end` — リレーを終了（追記不可、データは残る）
  - `/relay post <#channel>` — 全文を匿名で指定チャンネルに投稿（データは残る）
  - `/relay reveal <#channel>` — ネタバレ（執筆者一覧）を指定チャンネルに投稿（データは残る）
  - `/relay terminate` — リレーデータを削除
- 参加フロー: 「一文を追加」ボタン → モーダルで一文入力（前の一文がタイトルに表示）
- 連続投稿防止（同じユーザーが連続で書くのはNG）
- 2000文字超の場合は複数メッセージに自動分割

### 技術的変更

- `SESSION_KV` を共有利用（`relay-active:{guildId}` キープレフィックス、TTL 30日）
- deferred パターン（type:5 + waitUntil）でDiscord API呼び出しを非同期処理

## v0.9.0 — 検閲コマンド（ジョーク機能）

> 2026-03-21

### 追加

- メッセージコンテキストメニュー「検閲」 — 右クリック → Apps → 検閲でメッセージを削除し、1984風の演出で置き換え
  - Webhook で元の送信者の名前・アバターを使い「██████████ [検閲済み]」を投稿
  - Bot がジョージ・オーウェル『1984年』風のセリフをランダムで投稿（原作引用 + 通達系の2種類）
  - スレッド内でも動作（Webhook は親チャンネルに作成）
- `/censor-settings` コマンド — 検閲機能のモード設定（管理者のみ）
  - `全メッセージ対象`：誰のメッセージでも検閲可能（デフォルト）
  - `自分のメッセージのみ`：自分が送信したメッセージだけ検閲可能
  - `オフ`：検閲機能を無効化
- ManageMessages 権限のサーバーサイドチェック（コマンド表示設定の上書きによる悪用を防止）
