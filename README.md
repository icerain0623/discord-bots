## 自己紹介ワークフロー

チャンネルに常駐ボタンを設置し、ユーザーが4ステップのモーダルで自己紹介を入力してチャンネルに投稿できるワークフロー。

### フロー

```
「✏️ 自己紹介を書く」ボタン（チャンネル常駐）
    ↓
モーダル① 【基本①】名前・性別・年齢・肩書き・出身地        （本人のみ表示）
    ↓
モーダル② 【基本②＋好きな物①】趣味・特技・マイブーム・食べ物・飲み物
    ↓
モーダル③ 【好きな物②＋もっと①】場所・推し・音楽・本・いま欲しいもの
    ↓
モーダル④ 【もっと②＋一言】ペット・休日・返信・ゲーム・一言！
    ↓
最終確認（本人のみ）→「投稿する」ボタン
    ↓
チャンネルに自己紹介テキストを投稿
```

### 特徴

- 全ステップが本人にしか見えない（ephemeral）
- いつでもキャンセル可能（セッションは30分で自動破棄）
- 未入力フィールドは「未回答」として投稿

### セットアップコマンド

```bash
/setup-intro
```

任意のチャンネルで実行すると、そのチャンネルに「✏️ 自己紹介を書く」ボタン付きパネルメッセージを投稿します。

### 投稿フォーマット（出力例）

```
✨ @ユーザー名 さんの自己紹介 ✨

【基本】
名前：山田太郎
性別：男性
年齢：25
肩書き：エンジニア
出身地：東京都
趣味：プログラミング
特技：TypeScript
マイブーム：朝のコーヒー

【好きな物】
食べ物：ラーメン
飲み物：コーヒー
場所：秋葉原
推し・キャラクター：未回答
音楽：ロック
本：技術書

【もっと！】
いま欲しいもの：広いモニター
ペットを飼うなら：ねこ
休日はどう過ごす？：ゲーム
返信は早い？：早い
ゲームやってる？：やってる

【一言！】
よろしくお願いします！
```

## 技術スタック

- **Runtime:** Node.js 20+
- **ライブラリ:** discord.js v14
- **状態管理:** オンメモリ Map（TTL: 30分）
- **テスト:** Jest（ESM対応）

## プロジェクト構成

```
discord-bots/
├── src/
│   ├── index.js                   # Bot エントリー・イベントルーター
│   ├── deploy-commands.js         # スラッシュコマンド登録スクリプト
│   ├── commands/
│   │   └── setupIntro.js          # /setup-intro コマンド
│   ├── interactions/
│   │   ├── buttons.js             # ボタン操作ハンドラー
│   │   └── modals.js              # モーダル送信ハンドラー
│   ├── modals/
│   │   ├── modal1.js〜modal4.js   # モーダル定義
│   └── utils/
│       ├── sessionStore.js        # セッション管理（Map + TTL）
│       └── formatIntro.js         # 投稿テキスト整形
└── tests/
    ├── sessionStore.test.js
    └── formatIntro.test.js
```

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. 環境変数の設定

`.env.example` をコピーして `.env` を作成し、各値を設定します。

```bash
cp .env.example .env
```

| 変数名 | 説明 |
|--------|------|
| `DISCORD_TOKEN` | Bot のトークン（Discord Developer Portal で取得） |
| `CLIENT_ID` | Bot のアプリケーション ID |
| `GUILD_ID` | 開発・運用サーバーの ID |
| `INTRO_CHANNEL_ID` | 自己紹介を投稿するチャンネルの ID |

### 3. スラッシュコマンドを登録

```bash
npm run deploy
```

### 4. Bot を起動

```bash
npm start
```
