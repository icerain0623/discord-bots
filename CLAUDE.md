# CLAUDE.md

## デプロイ・push 時のルール

- コードの変更を push やデプロイする際は、以下のドキュメント更新が必要か確認し、ユーザーに提案すること
  - `README.md` — 機能説明、コマンド一覧、プロジェクト構成の更新
  - `RELEASE_NOTES.md` — 変更内容の記録
  - `package.json` — バージョン番号の更新（機能追加・破壊的変更がある場合）
- コードの変更を伴う push の場合は直接 main に push せず、PR を作成してからマージする

## バージョニングルール

セマンティックバージョニング（`MAJOR.MINOR.PATCH`）に従う。

- **MAJOR** — 破壊的変更（既存コマンドの削除・互換性のない仕様変更）
- **MINOR** — 機能追加（新コマンド、新しいワークフローなど）
- **PATCH** — バグ修正、ドキュメントのみの変更、リファクタリング

### バージョンを更新する箇所（すべて同時に更新すること）

1. `package.json` の `version` フィールド
2. `src/commands/status.js` の `VERSION` 定数
3. `RELEASE_NOTES.md` に新しいエントリを追加

### 注意事項

- 1つの MINOR バージョンに複数の機能をまとめない（機能ごとにバージョンを上げる）
- バージョン番号の重複は禁止（RELEASE_NOTES.md に同じバージョンが2つ以上あってはならない）
- ドキュメントのみの変更はバージョンを上げなくてよい
