# Matchup: 交流マッチング機能 設計書

## 概要

サーバーメンバー同士の交流を促進するマッチング機能。管理者が募集を開始し、参加者がトピックを選んで登録、管理者がマッチングを実行するとランダムにグループ分けされ、専用プライベートチャンネルで交流する。

## 利用シーン

既存メンバー同士で定期的に交流を促進するイベント（例：月1回のシャッフルマッチ）。

## コマンド構造

`/matchup` コマンドに以下のサブコマンドを持たせる。全て `Manage Guild` 権限が必要。

| サブコマンド | 説明 |
|---|---|
| `/matchup start` | 募集メッセージを投稿（参加ボタン付き） |
| `/matchup run` | 参加者をマッチング＆プライベートチャンネル作成 |
| `/matchup terminate` | 作成されたチャンネルを一括削除・イベント終了 |
| `/matchup topics add` | トピック追加 |
| `/matchup topics remove` | トピック削除 |
| `/matchup topics list` | トピック一覧表示 |

### コマンドオプション

**`/matchup start`**
- `group_size` (必須, integer): グループサイズ（2〜4）
- `category` (任意, channel): チャンネル作成先カテゴリ。未指定なら自動作成

**`/matchup run`** — オプションなし

**`/matchup terminate`** — オプションなし

**`/matchup topics add`**
- `name` (必須, string): 追加するトピック名

**`/matchup topics remove`**
- `name` (必須, string): 削除するトピック名

**`/matchup topics list`** — オプションなし

## 参加フロー

### 1. 募集開始

管理者が `/matchup start group_size:2` を実行すると、募集メッセージ（Embed）が投稿される：

```
🎲 交流マッチング募集中！
グループサイズ: 2人
現在の参加者: 0人

[参加する ✋]
```

### 2. 参加登録

ユーザーがボタンをクリックするとモーダルが開く：
- 定義済みトピックをセレクトメニューで複数選択
- 自由入力欄（フリーテキストで追加トピックを記入可能）

登録完了後、エフェメラルメッセージで確認：
```
✅ 参加登録しました！選択したトピック: ゲーム, 音楽, 「猫の話」
```

募集メッセージの参加者数はリアルタイム更新される。

### 3. 参加取消

同じボタンを再度クリックした場合：
- 「既に参加しています。取り消しますか？」とエフェメラルで確認ボタンを表示
- 取消後にトピックを変えて再参加も可能

## マッチング＆チャンネル作成

### マッチングロジック

- 参加者をランダムにシャッフル
- 指定された `group_size` でグループに分割
- 端数は最後のグループに吸収（例：7人で2人ペアの場合 → 2, 2, 3）

### チャンネル作成

専用カテゴリ内に各グループ用プライベートチャンネルを作成：
- チャンネル名: `matchup-1`, `matchup-2`, ...
- 権限: グループメンバー＋botのみ閲覧可能（`@everyone` は閲覧不可）

### 初回メッセージ

botがチャンネルに挨拶メッセージを投稿：

```
🎲 マッチングされました！
メンバー: @Alice @Bob

💬 共通の話題: ゲーム, 音楽
📝 Aliceさんのトピック: ゲーム, 音楽, 「猫の話」
📝 Bobさんのトピック: 音楽, 映画

楽しく交流してください！
```

共通トピックがない場合は共通の話題の行を省略し、それぞれのトピックだけ表示する。

### 募集メッセージ更新

マッチング完了後、募集メッセージを更新：
```
🎲 マッチング完了！
4グループが作成されました。
```

## 終了処理

`/matchup terminate` 実行時：
- イベントで作成された全チャンネルを一括削除
- 募集メッセージを更新：「🎲 このマッチングイベントは終了しました。」
- KVからイベントデータをクリーンアップ

### 将来的な拡張

- terminate 前にチャットログをエクスポートし、txt等のビューワーで閲覧可能にする機能（今回のスコープ外）

## データ設計（KV）

### `matchup-topics`
サーバーのトピック一覧（永続）。

```json
["ゲーム", "音楽", "映画", "アニメ", "料理"]
```

### `matchup-active`
現在アクティブなイベント情報（イベント中のみ存在）。

```json
{
  "messageId": "123456",
  "channelId": "789012",
  "groupSize": 2,
  "categoryId": "345678",
  "participants": [
    { "userId": "111", "topics": ["ゲーム", "音楽"], "freeTopics": ["猫の話"] },
    { "userId": "222", "topics": ["音楽", "映画"], "freeTopics": [] }
  ],
  "createdChannels": ["ch_id_1", "ch_id_2", "ch_id_3"]
}
```

### 制約
- アクティブなイベントは同時に1つのみ。`start` 時にアクティブがあればエラー。

## 状態遷移

```
なし → [start] → 募集中 → [run] → マッチング済 → [terminate] → なし
```

各コマンドは現在の状態を検証し、不正な遷移（例：募集中に terminate）はエラーメッセージを返す。

## トピック管理

- トピックはサーバーに1セット（グローバル）。イベントごとには分けない。
- `add`: 重複チェックあり
- `remove`: 存在しない場合はエラー
- `list`: 現在のトピック一覧をEmbed表示

## 技術的考慮事項

- 既存のCloudflare Workers + KVアーキテクチャに準拠
- インタラクション処理は既存の `worker.js` のルーティングに追加
- モーダル・ボタン処理は既存の `interactions/` パターンに従う
- Discord API呼び出し（チャンネル作成・削除・権限設定）は `utils/discordApi.js` を拡張
- チャンネル作成等の非同期処理が多いため、初回レスポンスで deferred message を返し、後続処理を行う
