# 1文リレー機能 設計書

## 概要

サーバーイベント用の1文リレー機能。各参加者が前の一文だけを見て次の文を追加し、最終的に完成した文章を投稿する。匿名で全文公開した後、任意のタイミングでネタバレ（執筆者一覧）を公開できる。

## コマンド一覧

すべてサーバー管理者（ManageGuild）のみ実行可能。サーバーにつき同時に1リレーのみ。

| コマンド | オプション | 説明 |
|---------|-----------|------|
| `/relay start` | `topic`(必須), `first_sentence`(必須) | お題と最初の一文を設定してリレー開始 |
| `/relay status` | なし | 全文を番号付きで表示（エフェメラル） |
| `/relay delete` | `number`(必須, Integer) | 指定番号の文を削除 |
| `/relay end` | `channel`(必須, Channel) | 全文（匿名）を指定チャンネルに投稿 |
| `/relay reveal` | `channel`(必須, Channel) | ネタバレを指定チャンネルに投稿、データ削除 |

## 参加フロー

1. 管理者が `/relay start` 実行
2. チャンネルにお題 + 「一文を追加」ボタンを含むメッセージが投稿される
3. ユーザーがボタンを押す
   - 直前の投稿者と同じ場合はエフェメラルでエラー表示
   - モーダルが表示される：前の一文がラベルに表示され、入力欄に新しい一文を入力
4. 送信後、KVに保存
5. チャンネルのメッセージが更新される（現在N文目の進捗表示）
6. 管理者が `/relay end #channel` で全文を匿名投稿
7. 管理者が `/relay reveal #channel` でネタバレ投稿、データ削除

## データ構造

### KV

キー: `relay-active:{guildId}` （TTLなし、管理者がrevealで明示的に削除）

```json
{
  "topic": "お題テキスト",
  "sentences": [
    {
      "text": "最初の一文",
      "userId": "123456789",
      "displayName": "管理者名"
    },
    {
      "text": "2番目の文",
      "userId": "987654321",
      "displayName": "ユーザー名"
    }
  ],
  "startedBy": "123456789",
  "startedAt": "2026-03-23T12:00:00.000Z"
}
```

## コマンド詳細

### `/relay start`

1. `relay-active:{guildId}` が存在するか確認。存在すればエラー（既にリレー進行中）
2. KVにデータ保存（topicと最初の一文をsentences[0]に格納。startedByは実行者）
3. チャンネルに以下のメッセージを投稿：
   - お題の表示
   - 「一文を追加」ボタン（customId: `relay_add`）
   - 進捗表示：「現在 1 文目」

### 「一文を追加」ボタン（`relay_add`）

1. KVからリレーデータを取得。存在しなければエラー
2. sentences配列の最後のuserIdと押したユーザーのIDを比較。同じならエフェメラルでエラー
3. モーダルを表示（customId: `relay_modal`）：
   - 前の一文をラベルとして表示
   - テキスト入力欄（Short、必須、最大200文字程度）

### モーダル送信（`relay_modal`）

1. KVからリレーデータを再取得（race condition対策）
2. 再度連続投稿チェック
3. sentences配列に新しい文を追加（text, userId, displayName）
4. KVに保存
5. エフェメラルで「追加しました！（N文目）」と応答
6. チャンネルのリレーメッセージを更新（進捗表示を更新）

### `/relay status`

1. KVからリレーデータを取得。存在しなければ「リレーは開催されていません」
2. エフェメラルで以下を表示：
   ```
   お題：〇〇（全N文）
   1. 一文目 — @ユーザーA
   2. 二文目 — @ユーザーB
   3. 三文目 — @ユーザーA
   ```

### `/relay delete`

1. KVからリレーデータを取得
2. 指定番号のバリデーション（1以上、sentences.length以下）
3. sentences配列から該当要素を削除
4. KVに保存
5. エフェメラルで「N番目の文を削除しました（残りM文）」と応答

### `/relay end`

1. KVからリレーデータを取得。存在しなければエラー
2. 指定チャンネルに全文を投稿：
   ```
   **お題：〇〇**

   一文目。二文目。三文目。...
   ```
3. KVデータは削除しない（revealのために保持）
4. エフェメラルで「投稿しました」と応答

### `/relay reveal`

1. KVからリレーデータを取得。存在しなければエラー
2. 指定チャンネルにネタバレを投稿：
   ```
   **お題：〇〇 — 執筆者一覧**

   1. 一文目 — @ユーザーA
   2. 二文目 — @ユーザーB
   3. 三文目 — @ユーザーA
   ```
3. KVデータを削除
4. エフェメラルで「ネタバレを投稿しました。リレーデータを削除しました」と応答

## エラーハンドリング

- リレー未開催時のend/reveal/delete/status → エフェメラルでエラー
- リレー開催中のstart → エフェメラルでエラー
- 連続投稿 → エフェメラルでエラー
- delete番号が範囲外 → エフェメラルでエラー
- KV読み書きのrace condition → モーダル送信時にKVを再取得して検証
